# 🤖 News Summary Automation

> **RSS + Make + OpenAI + Notion을 활용한 뉴스 요약 자동화 시스템**

---

# 📌 프로젝트 한눈에 보기

> **AI/IT 뉴스를 자동으로 수집 → AI가 요약 → Notion에 저장하는 자동화
> 프로젝트**

## 🎯 왜 만들었나요?

매일 새로운 AI 뉴스를 찾아보고 정리하는 과정은 반복적이고 시간이 많이
소요됩니다.

이 프로젝트는 **RSS + Make + OpenAI + Notion**을 연결하여 사람이 직접
복사·붙여넣기 하지 않아도 뉴스를 자동으로 관리할 수 있도록 만들었습니다.


### 🎯 프로젝트 목표

- RSS를 이용하여 최신 AI 뉴스를 자동으로 수집한다.
- AI를 활용하여 뉴스를 핵심 내용만 요약한다.
- 사람이 복사·붙여넣기 하지 않아도 Notion에 자동 저장한다.
- 동일한 뉴스가 여러 번 저장되지 않도록 중복을 방지한다.

---

# ✨ 주요 기능 (Features)

## 📰 1. RSS 자동 수집

**무엇을 하나요?**

(데이터 수집) RSS Feed에서 최신 AI 뉴스를 자동으로 가져옵니다. 몇개인지 적어야하나.?

**왜 필요한가요?**

사람이 뉴스 사이트를 매번 방문하지 않아도 최신 뉴스를 자동으로 수집할 수 있습니다.

---

## 📰 2. 뉴스 기사 수집/데이터 가공

**무엇을 하나요?**

(데이터 수집) 제목, 내용, 날짜 등 원하는 항목만 불러오기, 하나의 배열로 묶음

**왜 필요한가요?**

제공되는 수많은 RSS feed 필드 중 필요한 필드 즉 노션db에 들어갈 행만 뽑아서 가공하고 싶어서

---

## 🏷️ 3. AI 주제 필터링

**무엇을 하나요?**

(데이터 수집) AI 관련 키워드를 포함한 기사만 선택합니다.

**왜 필요한가요?**

원하지 않는 일반 뉴스는 제외하고 필요한 기사만 자동으로 가져오기 위해 사용합니다.

**사용 키워드**

| 키워드 | 설명 |
|---|---|
| AI agent | AI 에이전트 관련 기사 |
| LLM | 대형 언어모델 관련 기사 |
| generative AI | 생성형 AI 전반 |
| OpenAI | OpenAI 동향 및 제품 소식 |
| Claude | Anthropic / Claude 관련 |
| Gemini | Google AI 관련 |
| automation | 업무 자동화 사례 |
| enterprise AI | 기업 AI 도입 및 적용 사례 |

**키워드 선택 이유**

단순 기술 뉴스보다 **AI의 실질적 활용과 산업 영향**에 초점을 맞춰 키워드를 구성했습니다. 팀 프로젝트의 목적이 "자동화 파이프라인 학습"인 만큼, 자동화·에이전트·기업 도입 관련 기사가 학습 맥락과도 자연스럽게 연결됩니다.

**프롬프트**
```
json을 제공된 키워드들과 제일 연관이 있어보이는 아이템 하나를 json 배열에서 골라 선정해줘.
선정한 아이템이 있을 경우 Json 형태로 선택된 아이템만 출력해주고, 선택된 아이템이 없으면 빈 배열만 반환해줘
[키워드]
AI agent, LLM, generative AI, OpenAi, Claude, Gemini, automation, enterprise AI
[json]
{{5.array}}
```

---

## 📰 4. JSON 형태 변환

**무엇을 하나요?**

Json 형태로 변환합니다. 선정된 기사의 데이터 파일 형식이 json이라 명시해줌

**왜 필요한가요?**

다음 단계에서 make가 Notion에 url 데이터끼리의 중복 체크 요청을 내리기 위해서

---

## 📰 5. 노션 중복 검사

**무엇을 하나요?**

(비용/중복 방지) 노션 기능을 활용해 현재 선정된 기사가 기존 노션 페이지의 URL 텍스트값과 Dost not equl 상태인지 판별

**왜 필요한가요?**

기사 요약 과정 전에 토큰량을 불필요하게 소모하지 않기 위해 
현재 선정된 기사가 기존 노션 페이지에 중복되었는지 확인을 먼저 해야함

---

## 🤖 6. AI 자동 요약

**무엇을 하나요?**

(AI 가공) OpenAI (mini-5.4)를 이용하여 기사 내용을 3줄 이내로 요약합니다.

**왜 필요한가요?**

긴 기사를 모두 읽지 않아도 핵심 내용을 빠르게 확인할 수 있습니다.

**프롬프트**
```
첨부한 링크의 내용을 요약해줘. 
[출력 규칙]
1. 수식어 없이 요약 내용만 포함한다.
2. 최대 3줄을 넘어가지 않는다.
[url]
{{35.url}}
```

---

## 📝 7. Notion 자동 저장

**무엇을 하나요?**

(자동 저장) 요약 내용을 포함한 최종 결과를 Notion 데이터베이스에 자동 저장합니다.

**저장 항목**

- 제목
- AI 요약 (6단계의 결과.txt)
- 원문 링크
- 발행일

**왜 필요한가요?**

뉴스를 한곳에서 쉽게 검색하고 관리할 수 있습니다.

---

## 🔄 8. 예외 처리 (Retry)

**무엇을 하나요?**

(안정성) OpenAI 또는 Notion 호출이 실패하면 일정 시간 후 다시 시도합니다.

**현재 적용 정책**

| 상황 | 처리 방식 |
|------|-----------|
| OpenAI 호출 실패 | 10분 후 재시도 |
| Notion 저장 실패 | 10분 후 재시도 |
| 최대 재시도 횟수 | 2회 |

**왜 필요한가요?**

외부 서비스는 일시적으로 실패할 수 있기 때문에 자동으로 다시 시도하여 안정성을 높입니다.

# (보너스 미션) tags 내용 추가

- 프롬프트 내용까지 구상함
- 사용 프롬프트:
[tags 규칙]
tags는 글의 감성을 나타내는 키워드만 포함한다.
긍정·부정 감정을 모두 분석하여 해당하는 감성 태그를 작성한다.
태그는 3개 내외로 작성한다.
예시 태그: 긍정, 부정, 기쁨, 슬픔, 행복, 불안, 희망, 걱정, 만족, 실망, 기대, 후회 등.
내용에 없는 감성은 추측하여 추가하지 않는다.

# 💭 이번 프로젝트를 통해 배운 점

-   RSS는 뉴스를 자동으로 가져오는 역할을 한다. (API 호출 방법 외)
-   Make는 여러 서비스를 연결하여 자동화를 수행한다.
-   OpenAI는 뉴스 내용을 요약하는 역할을 한다.
-   Notion은 자동으로 정리된 데이터를 저장하는 공간이다.
-   자동화는 하나의 프로그램이 아니라 여러 서비스를 연결해 하나의
    흐름(Workflow)을 만드는 과정이라는 점을 이해했다.
